How WebServ Processes Requests

1. Event Loop

WebServ uses a single-threaded event loop based on epoll (Linux) or kqueue (BSD/macOS). This allows handling multiple connections simultaneously without the overhead of threads.

2. Config Parsing

Nginx-style configuration files are tokenized and parsed into a structured C++ object model, supporting multiple server blocks and location directives.

3. HTTP Strategy

We use the Strategy Pattern to handle different HTTP methods. The RequestHandler delegates execution to specific handlers (GetHandler, PostHandler, DeleteHandler).

4. CGI Interface

Dynamic content is handled by forking a child process and setting up pipes. Environment variables are passed according to the CGI 1.1 RFC.

Project Specs

  • Language: C++98 Standard
  • I/O Model: Non-blocking I/O with Multiplexing
  • Parsing: Custom State Machine HTTP Parser
  • No External Libraries (Boost, etc.)
